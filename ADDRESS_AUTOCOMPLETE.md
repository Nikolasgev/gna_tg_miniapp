# Автодополнение адресов

## Что реализовано:

1. ✅ **Backend endpoint** `/api/v1/addresses/suggest` - проксирует запросы к DaData API
2. ✅ **Flutter виджет** `AddressAutocompleteField` - поле ввода с автодополнением
3. ✅ **Интеграция в checkout** - расчет доставки только после выбора адреса из подсказок

## Как это работает:

### На фронтенде:

1. Пользователь начинает вводить адрес
2. После 2+ символов и задержки 300мс отправляется запрос на `/api/v1/addresses/suggest`
3. Показываются подсказки адресов в выпадающем списке
4. При выборе адреса из подсказок:
   - Адрес заполняется в поле
   - Вызывается `onAddressSelected` callback
   - Автоматически рассчитывается стоимость доставки

### На бэкенде:

- Endpoint проксирует запросы к DaData Suggest API
- Возвращает список подсказок с адресами и координатами (если доступны)

## Настройка:

### DaData API ключ (опционально):

Добавьте в `.env`:
```bash
DADATA_API_KEY=ваш_ключ_dadata
```

**Примечание:** DaData API работает и без ключа для базовых запросов, но с ограничениями. Для production рекомендуется получить бесплатный ключ на [dadata.ru](https://dadata.ru/api/suggest/address/).

## API Endpoint:

### GET `/api/v1/addresses/suggest`

**Параметры:**
- `query` (обязательный) - текст для поиска адресов
- `limit` (опциональный, по умолчанию 5) - максимальное количество подсказок (1-10)

**Пример запроса:**
```bash
curl "http://localhost:8000/api/v1/addresses/suggest?query=Москва, Тверская&limit=5"
```

**Пример ответа:**
```json
{
  "suggestions": [
    {
      "value": "г Москва, ул Тверская",
      "coordinates": [37.6064, 55.7558]
    },
    {
      "value": "г Москва, ул Тверская, д 1",
      "coordinates": [37.6065, 55.7559]
    }
  ]
}
```

## Преимущества:

✅ **Улучшенный UX** - пользователь видит подсказки при вводе  
✅ **Точность** - расчет доставки только для выбранного адреса  
✅ **Меньше запросов** - расчет доставки не вызывается при каждом изменении текста  
✅ **Координаты** - адреса могут содержать координаты для более точного расчета доставки
