# Настройка DaData API для автодополнения адресов

## Проблема

DaData API требует API ключ для работы. Без ключа возвращается ошибка 401 Unauthorized.

## Решение

### Вариант 1: Получить бесплатный API ключ DaData

1. Зарегистрируйтесь на [dadata.ru](https://dadata.ru/)
2. Перейдите в раздел [API ключи](https://dadata.ru/profile/#info)
3. Скопируйте ваш API ключ (Token)
4. Добавьте в `.env` файл:

```bash
DADATA_API_KEY=ваш_ключ_dadata
```

5. Перезапустите бэкенд:
```bash
cd backend
docker-compose restart backend
```

### Вариант 2: Использовать другой сервис

Можно заменить DaData на другой сервис автодополнения адресов, например:
- Яндекс Геокодер (тоже требует ключ)
- Google Places API (требует ключ)
- Собственный сервис с базой адресов

### Вариант 3: Отключить автодополнение

Если автодополнение не критично, можно использовать обычное текстовое поле без подсказок.

## Текущее состояние

- ✅ Виджет `AddressAutocompleteField` использует встроенный `Autocomplete` виджет Flutter
- ✅ Обработка ошибок добавлена - при отсутствии ключа просто не показываются подсказки
- ✅ Пользователь может ввести адрес вручную даже без подсказок

## Проверка работы

После добавления API ключа проверьте:

```bash
curl "http://localhost:8000/api/v1/addresses/suggest?query=Москва&limit=5"
```

Должен вернуться JSON с подсказками адресов.
